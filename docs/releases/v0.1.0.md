# v0.1.0

Initial public release of `dev-ai-toolkit`.

This release establishes the project as a local-first AI developer toolbox built with React, Vite, and TypeScript. The goal of `v0.1.0` is to provide a credible, runnable foundation with a few connected workflows already in place, while keeping the scope realistic for an early open-source release.

## What’s included

### Prompt workflows

The current release includes a prompt-centered workflow for creating, refining, and reusing prompt assets.

Included capabilities:

- Prompt template list, create, edit, duplicate, archive, restore, and delete flows
- Search and tag filtering for prompt templates
- Prompt template detail pages
- Prompt template import and export via JSON
- Prompt Playground with variable detection and live preview
- Prompt Diff for comparing prompt revisions
- Template version history with local restore support
- Prompt run snapshots and a dedicated run history page

### Developer utilities

The toolbox also includes a first batch of supporting developer tools:

- JSON Tools for formatting, validation, and minification
- API Builder for drafting request configurations and generating `fetch` snippets
- Code Viewer for reviewing text or code output in single or compare mode

### Product structure

This release also introduces a clearer application structure for new users:

- Overview page with grouped module presentation
- Navigation organized around `Prompt Workflows` and `Developer Utilities`
- A `Start here` path to highlight the primary workflow
- A local-first asset chain from template management to saved prompt runs

## Engineering foundations

To support ongoing maintenance, `v0.1.0` also includes:

- Feature-based project structure
- Local-first repository and service layers for prompt templates and prompt runs
- Versioned persisted payloads for browser-stored template and run data
- GitHub Actions CI running `lint`, `test`, and `build`
- Automated tests for core services, repositories, providers, and workflow behavior
- English and Simplified Chinese README files
- Basic contributor-facing project docs under `docs/`

## Known limitations

This is still an early release, and some constraints are intentional:

- All persistence is browser-based and local-first
- There is no backend or account system yet
- Saved runs are stored per browser and are not shared across devices
- Some utility modules are still lighter than the prompt workflow and could use deeper connections over time

## Roadmap

The next phase will focus on improving product coherence and maintainability rather than adding many more standalone pages.

### Near-term priorities

- Keep tightening the prompt workflow instead of adding many new standalone tools
- Expand workflow-level test coverage
- Keep local persistence behavior predictable as stored data evolves
- Improve maintainer-facing docs and deployment notes

### Longer-term priorities

- Better prompt asset lifecycle management
- Richer run history and output reuse
- Stronger maintainer-facing documentation
- Preparation for future API-backed or collaborative workflows

## Notes

`v0.1.0` should be treated as a stable starting point for iteration, not as a finished product. The current scope is intentionally modest: a few believable workflows, a clear code structure, and enough engineering discipline to support real maintenance without pretending the project is further along than it is.
